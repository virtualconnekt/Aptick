# Changelog

All notable changes to Aptick will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- TypeScript SDK for easy integration
- React hooks for frontend development
- Comprehensive documentation and examples

### Changed
- Improved error handling in smart contracts
- Enhanced UI/UX in frontend application

## [1.0.0] - 2024-01-XX

### Added
- Core billing smart contract on Aptos blockchain
- Provider registration system
- User escrow and deposit functionality
- Usage recording and automatic billing
- Service termination and refund system
- Next.js frontend application
- Aptos wallet integration (Petra, Martian, etc.)
- Real-time balance and usage tracking
- APT to octas conversion utilities

### Features
- **Smart Contract**
  - Provider registration with custom pricing
  - Secure escrow system for user deposits
  - Automatic payment processing for usage
  - Transparent refund mechanism
  - Gas-optimized operations

- **Frontend Application**
  - Provider registration interface
  - User deposit and balance management
  - Real-time transaction monitoring
  - Multi-wallet support
  - Responsive design with Tailwind CSS

- **Developer Tools**
  - Comprehensive Move unit tests
  - Development and deployment scripts
  - TypeScript type definitions
  - Error handling utilities

### Security
- Move language safety guarantees
- Resource-based asset management
- Access control for provider operations
- Input validation and sanitization

### Documentation
- Complete README with setup instructions
- API reference for smart contract functions
- Frontend integration examples
- Development guidelines

## [0.1.0] - 2024-01-XX (Initial Development)

### Added
- Initial project structure
- Basic Move smart contract skeleton
- Frontend boilerplate with Next.js
- Development environment setup
